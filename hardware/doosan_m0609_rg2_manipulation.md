## 주요 개념

### 1. TCP (Tool Center Point) 와 TCP Offset

- **TCP**: 로봇이 제어 기준으로 삼는 공구 끝단의 좌표점
- 로봇 기본 TCP는 플랜지 중심이지만, 그리퍼(RG2)를 장착하면 실제 파지점이 달라짐
- **TCP Offset**: 플랜지 기준에서 실제 공구 끝단까지의 거리/방향을 등록하는 값
- TCP Offset이 정확해야 작업 좌표계, 안전구역 판단이 정확하게 동작함

```
플랜지 중심 ──[TCP Offset]──▶ 실제 그리퍼 끝단 (TCP)
```

### 2. 안전구역 설정

- 티치 펜던트(TP)에서 설정하는 3D 공간 경계
- 로봇 TCP가 설정한 구역을 벗어나거나 침범하면 속도 제한 또는 정지
- **TCP Offset 값이 정확해야 안전구역 판단이 올바르게 동작함**
  - Offset이 틀리면 실제로는 안전구역 내에 있어도 판단이 어긋남
- 용도: 작업자 보호, 충돌 방지, 작업 영역 제한

### 3. 속도 제한과 "Short" 현상

- 로봇팔을 너무 빠른 속도로 동작시키면 모터 과전류/과부하 현상 발생 ("short"라고 표현)
- 이를 방지하기 위해 **소프트웨어 단에서 최대 속도를 제한**해두는 것이 일반적
- 속도 제한값을 코드로 강제하지 않으면 수강생/운용자가 임의로 해제할 수 있으므로 **감시 로직도 함께 구현**하는 것이 권장됨
- 관련 파라미터: `vel`, `acc` (두산 패키지 이동 함수의 속도/가속도 인자)

### 4. 싱귤러리티 (Singularity)

#### 개념

- 로봇팔이 특정 자세에 도달했을 때 **역기구학(IK)의 해가 무한하거나 존재하지 않는 상태**
- 수학적으로는 **야코비안(Jacobian) 행렬의 행렬식(determinant)이 0이 되는 상태**
- 이 상태에서 자유도(DoF)를 일시적으로 잃어 특정 방향 이동이 불가능해짐

#### 6축 로봇의 싱귤러리티 종류

| 종류 | 발생 조건 | 특징 |
|------|-----------|------|
| Wrist Singularity | 4축과 6축이 나란해질 때 (5축 = 0°) | 가장 흔함 |
| Shoulder Singularity | 손목 중심이 1축 회전축 위에 위치 | 팔이 뒤로 꺾이는 상황 |
| Elbow Singularity | 팔이 완전히 펴짐 (최대 도달 거리) | 팔꿈치 관절이 일직선 |

#### 움직이는 싱귤러리티 vs 안 움직이는 싱귤러리티

동일하게 싱귤러리티 상태라도 동작 여부가 다를 수 있음:

| 경우 | 설명 |
|------|------|
| **움직이는 경우** | 완전한 특이점이 아닌 **근방(near singularity)**이거나, 요청된 이동 방향이 자유도를 잃은 축 방향과 다를 때. 관절이 매우 빠르게 반응하며 불안정하게 움직일 수 있음 |
| **안 움직이는 경우** | **완전한 특이점**으로 IK 해가 무한하거나 없음. 로봇 컨트롤러가 이를 감지하고 안전을 위해 동작 자체를 거부 |

- 예시: `(0, 0, 90, 0, 90, 0)` 자세 → 싱귤러리티 근방이지만 이동 방향에 따라 동작 가능
- 계산으로 확인 가능: `det(J) ≈ 0` 이면 근방, `det(J) = 0` 이면 완전 특이점
- 실제 구현 시 **동작 명령 사이사이에 싱귤러리티 여부를 계산해서 예외처리**하는 것이 권장됨

#### 역학(Dynamics) 관점에서의 차이

기구학 관점(IK 해의 유무)과 별도로, **동역학 방정식** 측면에서도 두 경우는 다르게 해석된다.

로봇의 동역학 방정식:

```
τ = M(q)q̈ + C(q, q̇)q̇ + g(q)
```

- `τ`: 관절 토크 벡터
- `M(q)`: 관성(inertia) 행렬
- `C(q, q̇)q̇`: 코리올리/원심력 항
- `g(q)`: 중력 항

작업 공간(Cartesian)과 관절 공간(joint)을 연결하는 관계:

```
ẋ = J(q) · q̇          # 속도 관계
τ = Jᵀ(q) · F         # 힘-토크 관계 (정역학적)
```

**① 움직이는 경우 (near singularity) — 역학적 문제**

- `J`가 ill-conditioned 상태 → `J⁻¹` 또는 pseudoinverse가 매우 커짐
- 작업 공간에서 **작은 속도 명령**이 관절 속도 `q̇`를 폭발적으로 키움:
  ```
  q̇ = J⁺ · ẋ   →   ‖q̇‖ → ∞  (near singularity)
  ```
- 관절 속도가 급증 → `M(q)q̈` (관성력)과 `C(q,q̇)q̇` (코리올리력) 모두 폭증
- 결과: **모터에 순간적으로 매우 큰 토크가 요구됨** → 과전류/과부하 → "short" 현상
- 힘-토크 관점에서도: `τ = Jᵀ · F` 에서 `Jᵀ`가 ill-conditioned이면 **끝단에 작은 외력**이 가해져도 **관절 토크가 급격히 커짐**

**② 안 움직이는 경우 (exact singularity) — 역학적 해석**

- `det(J) = 0` → J의 rank 손실 → 특정 방향 자유도 소실
- 손실된 방향으로는 어떤 관절 토크를 가해도 **끝단이 해당 방향으로 힘을 낼 수 없음** (힘 전달 불가)
- 반대로: 끝단에서 해당 방향으로 외력이 가해지면 **관절에 전달되지 않고 기구학적으로 흡수됨** (관절 반력 없음)
- 컨트롤러는 `J⁻¹`을 계산할 수 없으므로 **토크 명령 자체를 생성하지 못해 동작 거부**
- Null space가 생기는 현상: 특정 관절 조합으로 움직여도 끝단은 정지 → 에너지가 관절 내부에서 소산

**요약 비교**

| 관점 | 움직이는 (near singularity) | 안 움직이는 (exact singularity) |
|------|-----------------------------|---------------------------------|
| Jacobian | ill-conditioned (det ≈ 0) | rank 손실 (det = 0) |
| 관절 속도 | 폭증 (q̇ → ∞) | 계산 불가 |
| 모터 토크 | 순간 과부하 (short 위험) | 토크 명령 생성 불가 → 동작 거부 |
| 힘 전달 | 증폭되어 관절에 전달 | 특정 방향 힘 전달 단절 |
| 코리올리/관성력 | 폭증 | 해당 방향 기여 없음 |

#### 싱귤러리티 회피 방법

두 가지 접근:

1. **`movejx` 사용** (두산 패키지 제공 함수)
   - 작업 공간(Cartesian)에서 목표점을 지정하되, 내부적으로 관절 공간 경로를 계획
   - 싱귤러리티를 우회하는 경로를 자동으로 선택
   - `movej`와 `movel`의 장점을 혼합한 형태

2. **싱귤러리티 구간에서 `movel`로 천천히 통과**
   - 속도를 매우 낮춰서 관절에 가해지는 급격한 토크 변화를 줄임
   - 싱귤러리티 완전 회피가 아닌 **안전한 통과** 전략
   - 속도: 일반 구간보다 현저히 낮게 설정 (예: 10% 이하)

```python
# 예시 (두산 패키지 기준 개념 코드)
# 싱귤러리티 구간 진입 전 속도 낮추기
movel(target_pos, vel=10, acc=10)   # 느린 속도로 통과

# movejx로 싱귤러리티 우회
movejx(target_pos, ref=DR_BASE)     # 관절 공간 경로 자동 계획
```

## 두산 로봇 패키지와 개발 철학

### dr 패키지 vs 직접 구현

- `dr_` 패키지: 두산이 공식 제공하는 ROS2 패키지 (티치 펜던트 연동, 기본 제어)
- 매니퓰레이터 응용 개발 시 dr 패키지를 그대로 쓰는 것만으로는 부족
- **실제 로봇 개발자는 패키지를 활용하는 것이 아니라 직접 만드는 것이 기본**

### 직접 패키지를 만들어야 하는 이유

1. 특수한 작업 시퀀스, 안전 로직, 센서 융합 등 커스텀 요구사항 대응
2. 기성 패키지는 범용 목적 → 특정 응용에 최적화 불가
3. 인터페이스(topic/service/action) 직접 설계 → 시스템 아키텍처 이해 필수
4. 실력 향상: 내부 동작 원리를 이해해야 문제 발생 시 디버깅 가능

### 개발 언어 선택

| 언어 | 특징 |
|------|------|
| Python | 빠른 프로토타이핑, 디버깅 용이 |
| C++ | 실시간 제어, 고성능 연산 필요 시 |
| ROS2 | 위 두 언어 모두 지원, 노드/토픽/서비스 인터페이스 |
